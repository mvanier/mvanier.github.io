CS 11: reference material

This page contains reference material that may be useful for students of CS 11 (Software Shop). The material described here ranges from elementary to very advanced.

General interest

books

  • Abelson and Sussman, Structure and Interpretation of Computer Programs

    The "Feynman Lectures" of computer programming. SICP (as it's known) is an extremely well-written guide to the fundamental ideas of programming. It is very dense; each page requires careful reading. The authors use the Scheme language (a dialect of lisp) to introduce programming concepts, and a bias towards functional programming styles is evident. Nevertheless, the big ideas are relevant to all programming languages, and in our opinion all serious programmers should read this book.

  • Essentials of Programming Languages, 2nd Ed. by Dan Friedman, Mitchell Wand, Christopher Haynes

    This book is the next step up after SICP. The authors create a series of programming languages in scheme which instantiate different language design choices. Good for advanced programmers with a serious interest in the implementation of computer languages.

  • Steve Macguire, Writing Solid Code

    This book contains a lot of useful information primarily aimed at C programmers. Especially useful are the chapters on the use of the assert() macro and on debugging.

  • Object-Oriented Software Construction, 2nd Ed. by Bertrand Meyer

    This book gives an exhaustive analysis of the principles of object-oriented programming. Meyer is the author of the Eiffel language, and his presentation is biased towards that world-view. However, OOSC contains a lot of useful information.

  • Gamma et. al. Design Patterns

    Design Patterns (also called the gang-of-four book or GOF after its four authors) describes a number of code patterns often found in object-oriented code which simplify many common tasks. The book is slanted towards C++ programmers, but the principles apply to all OO languages.

    A site with many Prolog links is here.

  • Martin Fowler, Refactoring

    Refactoring refers to the process of rewriting code to improve the design and/or implementation of some part of a program. It applies both to high-level design and to low-level implementation details. This book is a good introduction to the concept. More information is available at the author's web site.

  • Kent Beck, Extreme Programming Explained

    Many different software methodologies exist. Some of these are so heavyweight it feels like you need a congressional mandate in order to begin writing any code. Extreme Programming (XP) is a much lighter-weight methodology that is gaining increasing support. See also the author's web site.

web sites

  • The GNU home page.

    GNU (Gnu's Not Unix) is the name of the organization that originated the idea of free (as in speech, not as in beer) software. Founded and run by the always provocative Richard Stallman, GNU represents a philosophy of free exchange of software for the common good.

  • Eric Raymond's writings.

    Eric Raymond is a leading open source and Linux advocate (see this site for more on open source and its relation to free software in the GNU sense). His writings, especially "The Cathedral and the Bazaar", are interesting examinations of why open source development works as well as it does. He is rather full of himself, and like most advocates he tends to only see the positive side of his movement, but his writings are always interesting.

  • The jargon file.

    This is a somewhat Unix-centric collection of computer-related jargon. Very amusing when you get tired of doing real work.

  • Teach Yourself Programming in Ten Years.

    A useful antidote to the "Teach Yourself <language X> in 21 days/24 hours" books.

  • How to write unmaintainable code.

    A must-read for any programmer. It sounds preposterous, but most programmers run into code like this sooner or later. Mainly oriented to C/C++/java programs.

  • Esoteric Programming Languages.

    Think you've seen it all when it comes to programming languages? This site will cure you of that misconception. One particularly interesting language is unlambda. If you like this sort of thing, also check out the International Obfuscated C Code Contest.

C

C is a low-level imperative language originally designed for system programming as a replacement for assembly language. It has become ubiquitous both as a language for developing applications as well as a language for implementing other computer languages. However, use of C is on the decline as scripting languages and java are used more frequently. Nevertheless, a strong knowledge of C is essential for serious programmers.

books

  • Kernighan and Ritchie, The C Programming Language, 2nd Ed.

    A concise and complete description of the C language.

  • Peter van der Linden, Expert C Programming: Deep C Secrets

    This book, despite its numerous typos, is one of the few books that explains what is going on at the machine level during compilation and execution of a C program.

C++

C++ is an extension of C that supports object-oriented and generic programming. It is a comparatively difficult language to learn, for these reasons:
  • The language is very large and very complex.
  • There are many subtle pitfalls.
  • To master C++ you have to learn (at least) two new programming paradigms (object-oriented programming and generic programming). In particular, learning object-oriented programming requires learning a great deal about design issues.
  • Most compilers do not support the entire C++ standard at the moment.
  • Programmers who know C may be tempted to use C idioms even though there are better C++ idioms (e.g.using "char *" instead of "string" for strings.

However, C++ is a very powerful language which can be used effectively in cases where performance is critical.

books

  • Andrew Koenig and Barbara Moo, Accelerated C++

    This book is probably the best overall introduction to C++. However, it relies heavily on the standard template library and doesn't get around to object-oriented programming until comparatively late in the book.

  • Stan Lippman, Essential C++

    This book is a concise introduction to the major features of C++. Useful for someone who wants to pick up the main features of the language quickly.

  • Bjarne Stroustrup, The C++ Programming Language, Special Ed.

    This is the standard reference book by the author of the language. It will take you a long time to get through the whole thing. Covers the entire language. Not easy reading.

  • Stan Lippman and Josee Lajoie, The C++ Primer

    A good tutorial that also covers the entire language. Very long.

  • Scott Meyers, Effective C++ and More Effective C++

    Essential style guides for writing C++. If you want to avoid C++ pitfalls you must read these books. Meyers has also written a book called Essential STL which I have not read but which comes highly recommended.

  • Andrei Alexandrescu, Modern C++ Design: Generic Programming and Design Patterns Applied

    Not for the faint-hearted. This book explains some of the outer limits of C++, involving some amazing coding styles made possible by templates. Will make your head explode, but very cool.

  • Stan Lippman, Inside the C++ Object Model

    This book explains in detail what is going on at the machine level when a C++ program is compiled and run. Mainly for advanced programmers.

web sites

  • Bjarne Stroustrup's home page.

    Contains a very interesting FAQ on C++. Don't criticize C++ until you've read it.

  • A garbage collector for C and C++.

    Despite the fact that C and C++ are designed for manual memory management (with all the problems that that entails), it is possible to use automatic garbage collection as well. This page describes a freely-available implementation of a garbage collector which works with C and C++.

java

Java is a relatively new object-oriented language developed at Sun Microsystems. It is simpler than C++ and avoids the most problematic features of that language. The java environment is based around a portable bytecode and bytecode interpretation, which makes it extremely portable.

books

  • Gosling et al, The Java Programming Language

    This book explains the entire java language clearly and succinctly. It does not discuss the enormous java API, however.

web sites

  • The main Java web page at Sun Microsystems.

    This page has links to a variety of Java tutorials. You can also download the latest JDK here.

  • The Java API on-line reference.

Objective CAML (ocaml)

Objective CAML (ocaml) is a language which provides full support for functional programming and a strong, polymorphically-typed static type system. In practice, this means that the ocaml compiler catches a larger number of errors at compile time than almost any other language. The language features many advanced constructs such as variant records and built-in pattern matching, as well as an object system. Ocaml is also extremely fast, rivalling C speed for most tasks. Those who have used ocaml generally find it an extremely pleasant language to work in, despite the somewhat odd syntax.

books

  • Cousineau and Mauny, The Functional Approach to Programming

    This book is, to our knowledge, the only English-language book on Caml. It does not use Objective Caml but an earlier dialect called Caml light. However, the differences are minor. The book is excellent, but its main focus is on teaching functional programming rather than Caml programming per se.

web sites

  • The ocaml home page.

  • Yet another ocaml home page.

  • Documentation for the latest ocaml release.

scheme and lisp

Scheme and lisp are languages that feature a parenthesized prefix syntax and a mostly-functional programming paradigm. Historically, lisp has mainly been used for artificial intelligence applications while scheme has mainly been a language used to teach programming concepts. Both languages can be used outside these domains, however.

books

  • SICP, as mentioned above.

  • Scheme and the Art of Programming by George Springer and Dan Friedman.
  • Simply Scheme by Brian Harvey and Matthew Wright.

    These last two books are a more gentle introduction to scheme than SICP.

  • The Little Schemer and The Seasoned Schemer by Dan Friedman and Matthias Felleisen.

    These two books are short introductions to scheme written in a question-and-answer format. They teach how to think about problems in a functional/recursive way, and are very useful as a companion to SICP, for instance.

  • ANSI Common Lisp and On Lisp, by Paul Graham.

    These two books describe Common Lisp, which is less elegant than scheme but more practical for serious programming tasks (and much faster if used correctly). ANSI Common Lisp is a very thorough description of the language, while On Lisp is (among other things) the most complete introduction to Lisp macros available anywhere. Lisp macros allow you to customize lisp syntax to your application, and are one of the amazing features of lisp.

web sites

  • The scheme home page at MIT.

  • MIT scheme reference material.

  • PLT scheme.

    Another solid implementation of scheme. Home of the well-known DrScheme implementation. PLT software can be downloaded here.

  • Teach Yourself Scheme in Fixnum Days.

    A nice tutorial on scheme.

  • The Internet Scheme Repository.

    Tons of scheme-related stuff. I especially like the list of publications.

  • Steel Bank Common Lisp

    Probably the best open-source common lisp implementation.

python

Python is probably the best "scripting language" available today. Scripting languages emphasize ease of programming, rapid development, and easy connection to libraries written in C, C++, and java. Python also has the advantages of being object-oriented, extremely readable, cross-platform, and extremely easy to learn.

books

  • Learning Python by Mark Lutz (O'Reilly and Associates)

    A clearly-written tutorial.

web sites

  • The Python home page.

    The main site to look for information on python.

  • The comp.lang.python news group.

  • The Jython home page.

    Jython is an implementation of python written entirely in java, and providing easy scripting of java components.

Other interesting languages

  • Matlab

    Matlab is far from elegant as a programming environment, but it is an indispensable tool for most scientists. The programming language uses two-dimensional arrays (matrices) as the primary data types, and most operations can work on both scalars and matrices.

  • Mathematica

    Mathematica is the primary computer algebra system in use today. Its programming language borrows heavily from Lisp and many other languages.

  • Fortran

    Fortran is an extremely archaic language which survives because of the large amount of scientific and engineering code written in it. Some Fortran compilers can produce extremely efficient code (faster than C in some cases), and many scientists (unfortunately) do all of their work in Fortran. Learning to program in Fortran can create bad mental habits that are extremely difficult to overcome, and those who learn to program in Fortran first often have great difficulties learning other computer languages.

  • C#

    C# is a new language from Microsoft, which is being positioned as a competitor to java. Despite its source, the language appears to be well-designed and improves on java in a number of respects.

  • Eiffel

    Eiffel is a pure object-oriented language with an advanced assertion system known as Design by Contract that facilitates the design of correct software, as well as a number of other features. The canonical Eiffel site is here. A good free Eiffel compiler is the GNU SmallEiffel compiler.

  • Haskell

    Haskell is a so-called lazy functional language. Programs in Haskell do not use any assignment statements whatsoever. Reasoning about programs is particularly easy in Haskell, though the lack of imperative features makes the language challenging to learn.

  • Standard ML

    Standard ML is quite similar to ocaml. It has a nicer syntax but is not as practical for most uses (and is slower).

  • Smalltalk

    Smalltalk is the earliest of the object-oriented programming languages still in use today. It has a small but fanatical following. Programming in Smalltalk is quite different from programming in Java or C++. Smalltalk is dynamically-typed, and emphasizes interactive development and easy modifiability. The language itself is incredibly simple. Squeak is an interesting recent implementation of Smalltalk.

  • Prolog

    Prolog is a logic programming language. In such languages, you program by describing the problem to be solved rather than by describing a series of steps to solve it, and the computer figures out how to solve it. One free implementation is GNU prolog. A site with many Prolog links is here. Another interesting logic programming language is miniKanren. A logic programming language with static typing is Mercury.

  • Perl

    Perl is a scripting language widely used for web development. As a language, perl is very powerful but violates nearly every rule of good programming language design. Perl code is typically extremely difficult to decipher after it's written. We believe that Python is a superior alternative to perl for almost all tasks for which perl is used.

  • tcl

    Tcl ("tool command language") is a simple scripting language with a very nice graphical toolkit called Tk. Tcl is inferior to python for scripting tasks, so its use is not recommended.

  • Forth

    Forth is a stack-based threaded interpreted language with an unusual postfix syntax. Forth is interesting from a language design standpoint but is quite cumbersome to use and tends to lead to unreadable code. Forth is really only useful in embedded applications. A good free implementation of Forth is gforth.

  • Dylan

    Dylan is a language which is very similar to scheme or lisp but which has an infix syntax. A free implementation is Gwydion Dylan.

  • J

    J is a dialect of APL, an array-based programming language which was infamous for requiring a strange set of characters and which typically resulted in incredibly short (and incomprehensible) programs. J uses ascii characters only, and is more functional than APL, but code written in J is still basically write-only.
