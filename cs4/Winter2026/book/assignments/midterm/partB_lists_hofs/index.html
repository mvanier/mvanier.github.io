    Part B: Lists and higher-order functions (30 points)

    In this section, you'll write some functions involving lists and some higher-order functions.

    Warning

    Don't use any OCaml library functions to write these functions unless we explicitly say it's OK.

    The one exception to this is the List.rev function, which you are allowed to use anywhere.

    The theme of this section is increasingly elaborate and abstract sorting functions.

    1. split2   [3 points]

    We are going to be re-creating the merge sort function we explored in the lectures. We'll start by writing a function called split2 which takes a list and splits it into two equal halves (or as nearly equal as we can get).

    In constrast to the splitting functions described in the lectures, this function:

    • will be iterative,
    • will only do a single pass through the input list,
    • and will assign successive values in the input list to the first and second output lists, alternating between them.

    If there are an odd number of elements in the input list, the first output list will get one more element than the second output list.

    Asymptotic time complexity: \(\Theta(N)\) where \(N\) is the length of the input list.

    Examples

    # split2 [] ;;
- : int list * int list = ([], [])

# split2 [0] ;;
- : int list * int list = ([0], [])

# split2 [1; 1] ;;
- : int list * int list = ([1], [1])

# split2 [1; 2] ;;
- : int list * int list = ([1], [2])

# split2 [12; 7; 0] ;;
- : int list * int list = ([12; 0], [7])

# split2 [6; 14; 14] ;;
- : int list * int list = ([6; 14], [14])

# split2 [6; 1; 4; 4] ;;
- : int list * int list = ([6; 4], [1; 4])

# split2 [12; 7; 13; 9] ;;
- : int list * int list = ([12; 13], [7; 9])

# split2 [13; 11; 0; 14; 11] ;;
- : int list * int list = ([13; 0; 11], [11; 14])

# split2 [14; 13; 14; 4; 2; 10; 1; 12; 14; 11] ;;
- : int list * int list = ([14; 14; 2; 1; 14], [13; 4; 10; 12; 11])

    Write this function in an iterative (tail-recursive) way. (The split function itself doesn't need to be recursive, but there needs to be a tail-recursive helper function.)

    2. merge2   [3 points]

    The next part of a merge sort function is the merge function. Write a function called merge2 which merges two sorted lists to make one larger list containing all of (and only) the elements of the two input lists in sorted (ascending) order. It does this by picking off the smaller value of the heads of each list, making that the next element of the output list, and recursing on the other parts of the input lists.

    In constrast to the function described in the lectures, this function will only merge in ascending order, and it must be iterative.

    Asymptotic time complexity: \(\Theta(N)\) where \(N\) is the sum of the lengths of the input lists.

    Examples

    # merge2 [] [] ;;
- : int list = []
# merge2 [0] [] ;;
- : int list = [0]
# merge2 [1] [1] ;;
- : int list = [1; 1]
# merge2 [1] [2] ;;
- : int list = [1; 2]
# merge2 [0; 12] [7] ;;
- : int list = [0; 7; 12]
# merge2 [6; 14] [14] ;;
- : int list = [6; 14; 14]
# merge2 [4; 6] [1; 4] ;;
- : int list = [1; 4; 4; 6]
# merge2 [12; 13] [7; 9] ;;
- : int list = [7; 9; 12; 13]
# merge2 [0; 11; 13] [11; 14] ;;
- : int list = [0; 11; 11; 13; 14]
# merge2 [1; 2; 14; 14; 14] [4; 10; 11; 12; 13] ;;
- : int list = [1; 2; 4; 10; 11; 12; 13; 14; 14; 14]

    Tip

    Watch out for the base cases! Whenever one input list is empty, you should immediately finish the merge without doing any more recursive calls to merge2. (This is a requirement of this problem.) Hint: the List.rev function is your friend!

    3. split3   [4 points]

    Write a function called split3 which is exactly like split2 except that it splits the input list into three equal lists (or as nearly equal as is possible).

    Like split2, this function:

    • will be iterative,
    • and will only do a single pass through the input list.

    It will assign successive values in the input list to the first, second, and third output lists, alternating between them in that order.

    If the number of elements in the input list is not divisible by 3, the first (and maybe the second) output list(s) will get the extra elements.

    Asymptotic time complexity: \(\Theta(N)\), where \(N\) is the length of the input list.

    Examples

    # split3 [] ;;
- : int list * int list * int list = ([], [], [])

# split3 [0] ;;
- : int list * int list * int list = ([0], [], [])

# split3 [1; 1] ;;
- : int list * int list * int list = ([1], [1], [])

# split3 [1; 2] ;;
- : int list * int list * int list = ([1], [2], [])

# split3 [12; 7; 0] ;;
- : int list * int list * int list = ([12], [7], [0])

# split3 [6; 14; 14] ;;
- : int list * int list * int list = ([6], [14], [14])

# split3 [6; 1; 4; 4] ;;
- : int list * int list * int list = ([6; 4], [1], [4])

# split3 [12; 7; 13; 9] ;;
- : int list * int list * int list = ([12; 9], [7], [13])

# split3 [13; 11; 0; 14; 11] ;;
- : int list * int list * int list = ([13; 14], [11; 11], [0])

# split3 [14; 13; 14; 4; 2; 10; 1; 12; 14; 11] ;;
- : int list * int list * int list = ([14; 4; 1; 11], [13; 2; 12], [14; 10; 14])

    4. merge3   [4 points]

    Write a function called merge3 that will merge three sorted lists into one sorted list. (All lists are sorted in ascending order.) Do not do this by using merge2 on two of the lists, and passing the result and the remaining list to merge2. Instead, your function must merge the three input lists in a single pass through them.

    Asymptotic time complexity: \(\Theta(N)\), where \(N\) is the sum of the lengths of the input lists.

    Like merge2, this function will merge in ascending order, and it must be iterative.

    Tips

    In this function, there are four base cases: where all of the lists are empty (one case), or where any two of the three lists are empty (three cases). In these cases, you should finish the merge without having to do any more recursive calls to merge3. (This is a requirement of this problem.)

    Also, when only one of the three lists are empty, you are permitted (but not required) to use the merge2 function you defined above to complete the merge without having to do any further recursive calls to merge3. This is the only use of merge2 that is allowed in the function.

    Examples

    # merge3 [] [] [] ;;
- : int list = []

# merge3 [0] [] [] ;;
- : int list = [0]

# merge3 [1] [1] [] ;;
- : int list = [1; 1]

# merge3 [1] [2] [] ;;
- : int list = [1; 2]

# merge3 [12] [7] [0] ;;
- : int list = [0; 7; 12]

# merge3 [6] [14] [14] ;;
- : int list = [6; 14; 14]

# merge3 [4; 6] [1] [4] ;;
- : int list = [1; 4; 4; 6]

# merge3 [9; 12] [7] [13] ;;
- : int list = [7; 9; 12; 13]

# merge3 [13; 14] [11; 11] [0] ;;
- : int list = [0; 11; 11; 13; 14]

# merge3 [1; 4; 11; 14] [2; 12; 13] [10; 14; 14] ;;
- : int list = [1; 2; 4; 10; 11; 12; 13; 14; 14; 14]

    5. mergesort3   [3 points]

    Now that split3 and merge3 are defined, put them together into a function called mergesort3 that sorts an input list in ascending order by breaking it into three equal parts, sorting the parts, and merging the results.

    This function does not need to be iterative, and it's much easier to define as a recursive process.

    Examples

    # mergesort3 [] ;;
- : int list = []

# mergesort3 [0] ;;
- : int list = [0]

# mergesort3 [1; 1] ;;
- : int list = [1; 1]

# mergesort3 [1; 2] ;;
- : int list = [1; 2]

# mergesort3 [12; 7; 0] ;;
- : int list = [0; 7; 12]

# mergesort3 [6; 14; 14] ;;
- : int list = [6; 14; 14]

# mergesort3 [6; 1; 4; 4] ;;
- : int list = [1; 4; 4; 6]

# mergesort3 [12; 7; 13; 9] ;;
- : int list = [7; 9; 12; 13]

# mergesort3 [13; 11; 0; 14; 11] ;;
- : int list = [0; 11; 11; 13; 14]

# mergesort3 [14; 13; 14; 4; 2; 10; 1; 12; 14; 11] ;;
- : int list = [1; 2; 4; 10; 11; 12; 13; 14; 14; 14]

    Generating sorting functions (genrec)

    Note

    There is nothing to hand in for this subsection, but the material discussed is essential in solving the remaining problems, so read on!

    One of the themes of this course is that any recursive pattern can be encapsulated in a higher-order function. You've seen mergesort in the lectures, quicksort in the assignments, and you've implemented mergesort3 above. Can we encapsulate all their recursion patterns using a single higher-order function? The answer is yes!

    For the remaining problems, you will be using this higher-order function:

    let rec genrec base split combine lst =
  if base lst then
    lst
  else
    let (preserve, recurse) = split lst in
    let recursed = List.map (genrec base split combine) recurse in
      combine preserve recursed

    This encapsulates the recursion patterns found in both mergesort and quicksort functions:

    • base is a function which returns true when the input list to be sorted is in one of its base case configurations (and thus, doesn't need to be sorted).

    • split is a function which splits the input list. It returns two lists:

      1. preserve : a list of elements of the input list which are to pass through unaltered (for instance, the pivot value in a quicksort function). If this isn't needed (e.g. in a merge sort function), this will be an empty list.

      2. recurse : a list of lists, each of which was generated from the input list. These lists will get processed by recursive calls to genrec. There could be any number of sublists, depending on the algorithm being implemented. For instance, simple merge sort and quicksort would have two sublists, whereas mergesort3 would have three.

    • combine is a function which combines the preserved values with the recursed lists (after they have been recursively processed) to get the final result.

    Using genrec means that we can generete sorting functions simply by specifying the base case function, the splitting function, and the combining function. The base case function is pretty trivial. The splitting and combining functions may be simple or complex, depending on the algorithm. However, since we've implemented a few of these above, we can re-use them here!

    6. Generating mergesort   [3 points]

    Use genrec to generate a simple mergesort function that splits the input list into two lists, sorts the lists, and merges the sorted sublists. Use the split2 and merge2 functions to create the input splitting (split) and merging (combine) function arguments to genrec.

    Tips

    You can't use split2 and merge2 directly as the split and combine arguments, but the actual split and combine functions you need can be defined easily in terms of split2 and merge2, respectively.

    Remember that both the split and combine functions allow for preserved values, though in the case of merge sort, there aren't any. So the split function will need to return the empty list as the preserve output, and the combine function will receive the empty list as the preserve input.

    In the combine function, when unpacking the recursed argument (a list of lists) there will be cases that can't happen; in this case use the failwith command with a meaningful error message to indicate this. (This applies to the problems below as well.)

    The sorting examples are the same as in problem 5.

    7. Generating mergesort3   [3 points]

    Use genrec to generate a mergesort3-like function; call it mergesort3' (note the single quote (') at the end of the name). Use the split3 function you defined above to define the split function argument to genrec; as described above, it won't work as-is, so you will need to adjust its output to be compatible with what split is supposed to return. Similarly, use the merge3 function to define the combine function argument to genrec; again, you can't use it as-is.

    The sorting examples are the same as in problem 5.

    8. Generating quicksort   [3 points]

    Now use genrec to generate a quicksort function. Recall that the quicksort algorithm works as follows:

    1. If the list has a length <= 1, return it unchanged.

    2. Otherwise, take the first element (head) of the list (called the "pivot"), and set it aside.

    3. Take the tail of the list, and split it into two sublists:

      1. all elements strictly less than the pivot
      2. all elements greater than or equal to the pivot

    4. Sort the sublists, then combine

      1. the sorted less-than sublist
      2. the pivot
      3. the sorted greater-than-or-equal sublist

      to get the final answer.

    Use the same base function you've been using for the previous two problems.

    You will have to define the split function yourself (don't use any previously-defined functions); note that the preserved output list will not be empty. (Hint: It just contains the pivot.) If there are any input cases you can't split (for instance, an empty list), use the failwith function and a meaningful error message to handle those cases.

    Use the List.filter function to easily generate the sublists. (Look it up in the OCaml List module documentation.) You must also define the combine function yourself, based on how the quicksort algorithm works.

    The sorting examples are the same as in problem 5.

    9. Generating quicksort3   [4 points]

    We extended mergesort to mergesort3 above. Can we do the same to quicksort? Indeed we can!

    Here's the algorithm:

    1. If the list has a length <= 1, return it unchanged.

    2. Otherwise, take the first two elements (head1 and head2) of the list (called the "pivots"), and set them aside. Order them by size; the smaller is pivot_small and the larger is pivot_large. If they are equal, both values will be the same (that still works).

    3. Take the rest of the list, and split it into three sublists:

      1. all elements strictly less than pivot_small
      2. all elements greater than or equal to pivot_small but strictly less than pivot_large
      3. all elements greater than or equal to pivot_large

    4. Sort the sublists, then combine

      1. the sorted sublist which is less than pivot_small
      2. pivot_small
      3. the sorted sublist which is between pivot_small and pivot_large
      4. pivot_large
      5. the sorted sublist which is greater than or equal to pivot_large

      to get the final answer.

    Once again, use genrec to write the function. Despite the increase in complexity, this only requires a couple of lines more than the quicksort function did. Again, use List.filter to generate the sublists.

    The sorting examples are the same as in problem 5.
