    Part A: Imperative programming (20 points)

    In this section, we are going to write a few functions using OCaml's imperative features.

    1. Fibonacci again

    [10 points]

    Write two imperative versions of the fibonacci function that takes a single int argument n and returns an int result, which is the nth fibonacci number. The first one should be called fibonacci and should use a while loop, while the second should be called fibonacci2 and should use a for loop. Do not use recursion or any helper functions in either problem. The resulting functions should have \(\Theta(n)\) worst-case asymptotic time complexity. The functions should store all data in reference cells (except for the argument n and loop indices in fibonacci2) and update them as necessary.

    Hint

    Write these functions basically the way you would write them in C or Python (modulo syntax, of course).

    2. Selection sort

    [10 points]

    In this problem, you will write a function called selection_sort which implements an imperative sorting algorithm that works on arrays of values, sorting in-place in ascending order using the "selection sort" algorithm. This function has the signature:

    val selection_sort : 'a array -> unit

    You may assume that the type 'a is orderable i.e. that operators like <, =, and > work properly on values of that type.

    Again, do not use recursion or any helper functions in your function. Use for loops or while loops for looping.

    Selection sort works as follows.

    • If the length of the array is less than 2, do nothing. (The array is already sorted.)
    • Otherwise, start at the beginning of the array. Set the "current index" to 0 (the first location in the array).
    • Loop through every value (using either a while or a for loop) starting from the current index and continuing to the end, looking for the smallest value. When you find it, save it (e.g. into a ref cell). Continue to the end of the array, since there may be even smaller values later in the array.
    • At the end of each iteration of the loop, swap the smallest value with the value at the current index.
    • Repeat with the current index incremented by 1.
    • Stop when the current index is larger than the last index in the array. (If you use a for loop, the loop will automatically stop at this point.)

    At the end of this process, the array will be sorted. This algorithm has a time complexity of \(\Theta(n^2)\).

    You don't need to use any helper functions for this problem, and you shouldn't use any.

    Notes

    • You will need to have two nested loops: the inner one to find the minimum values, and the outer one to advance the current index.

    • Note that for loop indices are inclusive; for i = 1 to 10 do ... done loops through all indices from 1 up to and including 10.

    • You can have an if expression without an else clause in OCaml as long as the type of the then clause is unit (see the ocaml_cheat_sheet.)
