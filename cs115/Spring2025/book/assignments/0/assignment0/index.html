    Assignment 0: Getting set up

    Due date

    This assignment has no due date, and isn't graded.

    Overview

    The purpose of this pseudo-assignment is to make sure that all the software you need in order to do the assignments for this class is set up correctly. This is not a real assignment and it's not worth any marks, but if you want to complete the actual assignments you'll need to do it.

    Warning

    You are responsible for getting the course software installed. If you are having problems with this, the course staff is here to assist you. DO NOT write your assignments in an online Haskell environment, even though this may work for some of the early assignments. It will not work for later assignments, so if you try to do this, you are likely to get heavily penalized.

    Getting Haskell installed

    See the software installation page for detailed instructions on how to install the course software.

    Using Haskell

    There are only a few things you need to know in order to use Haskell effectively.

    Starting Haskell

    Open a terminal and type:

    $ ghci

    at the terminal prompt. (The $ is the terminal prompt; don't type that.) This will bring up the Haskell interactive interpreter, which is a good environment for experimenting with the language and for testing code you've written. (This is similar to the Python interactive interpreter.)

    ghci makes it possible to easily recall and edit previously-input lines of text by using the up and down arrow keys. To see what it can do, start up ghci:

    $ ghci

    and type the following commands (one per line, hitting the return key at the end of each line):

    ghci> putStrLn "hello"
ghci> putStrLn "goodbye"

    Note

    The ghci prompt is ghci>; don't type that!

    These lines should, when evaluated, print the words "hello" and "goodbye" respectively. Once you've done this, you should be able to recall either line by using the up arrow key. For instance, you can hit the up arrow key once to get the line:

    putStrLn "goodbye"

    and edit it so that it says:

    putStrLn "hasta la vista"

    When you hit return, it should print out "hasta la vista" on a separate line. This feature makes line editing much easier. You can also use control-a (hold the control key down and type a) to get to the beginning of a line you are editing and control-e to get to the end. control-l (lower-case L) clears the terminal and puts the cursor at the top of the terminal window.

    Exit the interpreter by typing control-d (this is just like Python).

    ghci: multiline input

    ghci normally works one line at a time, like the Python interpreter does. However, it's possible to get it to accept multiline input as well. To do this, enter the characters :{ at the ghci prompt:

    ghci> :{

    This puts you into multiline input mode. ghci will respond with this:

    ghci|

    and allow you to type any number of lines without evaluating them. When you are done, type :} to terminate multiline input mode, and ghci will evaluate the code you just input and go back to the regular one-line-at-a-time mode.

    Here's a trivial example:

    ghci> :{
ghci| increment :: Int -> Int
ghci| increment x = x + 1
ghci| :}
ghci>

    Multiline input is very convenient when entering functions that consist of multiple equations, and/or when you want to include the type signature with the function.

    Installing libraries with cabal

    The Haskell package manager is called cabal. We will use it to install libraries that we need. Instructions on how to do this will be included with the assignments that use the libraries.

    Compiling stand-alone executables

    For the first few assignments, we will mainly use the interactive interpreter (ghci). Starting with assignment 4, we will start compiling standalone programs. Instructions on how to do this will be included in assignments that use this.

    Using Visual Studio Code

    You can use any text editor you like to write Haskell code, but we recommend you try Visual Studio Code (also known as VS Code), which has excellent Haskell support, and isn't difficult to set up.1 In order to use it, you need to do the following steps:

    • Install the Haskell Language Server (HLS).

      The software installation page contains instructions on how to do this. Language servers provide a way for editors to query languages so that things like command completion and type information can be displayed in the editor while you're editing code.

    • Install Visual Studio Code from the VS Code website.

    • When inside VS Code, look at the Extensions pane (select View/Extensions from the menu). Type "Haskell" into the search bar at the top, and select and install "Haskell Platform".

    • If code hinting software (Copilot, ChatGPT etc.) comes pre-installed with VS Code, switch it off! If you can't figure out how, ask the instructor and/or the TAs.

    Now you will get nice syntax highlighting and code completion when you edit Haskell source code. You can also start a terminal while inside VS Code to test your code by running the Haskell interpreter.

    OK, now you are ready to write Haskell code!

    1. I (Mike) actually use Neovim as my code editor most of the time. Neovim is incredibly powerful, but it takes a lot longer to learn and to configure. I don't recommend it to students unless they are willing to spend a lot of time learning it. ↩
